# 프로그램 사용 전, 초기 설정

## STEP 0. 기본 동작법 (터미널 기반)

### 1. 터미널 열기

터미널을 기반으로 작동하므로 터미널 열기

### 2. 파일 위치로 이동

cd 명령어를 통해 diary.sh가 있는 디렉토리로 이동

### 3. 프로그램 최초 실행

./diary.sh 명령어를 이용하여 프로그램 실행  
<font color = "red"> 만약, 실행 권한이 거절된다면 chmod +x diary.sh 명령어 입력 </font>

## STEP 1. 시작하기 (초기설정)

### 1. 소프트웨어 설치 (Install)

기능: 터미널 어디서든 diary 명령어만으로 프로그램을 실행할 수 있도록 단축 명령어를 등록  

동작: 설치가 완료되면 설정 적용을 위해 사용자의 쉘(Shell)이 자동으로 재시작. 이후 ./ 경로 입력 없이 diary만 입력하여 실행

### 2. Github 연결 설정 (Connect)

기능: 일기 데이터를 저장할 Github 계정과 연동

동작:

현재 폴더를 Git 저장소로 초기화

사용자의 Github ID와 Repository(저장소) 주소를 입력받음

보안을 위해 Token(인증 토큰) 입력 시 화면에 문자가 표시되지 않음

------------------------------------------------------------

## STEP 2. 일기 사용하기 (기본 기능)

### 3. 일기 쓰기 (Write)

날짜와 내용을 입력한 후, 개인 암호키를 설정

데이터는 즉시 암호화(Encryption)되어 .enc 확장자 파일로 저장

### 4. 일기 조회 (Read)

파일 목록에서 조회할 날짜를 선택

저장 시 설정한 암호키를 입력. 키가 일치할 경우 내용을 복호화하여 화면에 표시

복호화(Decryption): 암호화된 데이터를 다시 읽을 수 있는 상태로 변환하는 것.

### 5. 일기 수정 (Edit)

수정할 파일 선택 후 암호키를 입력하여 인증

내용 수정이 완료되면, 즉시 재암호화하여 원본 파일에 덮어쓰기

보안 유지를 위해 수정 과정에서 생성된 평문(암호화되지 않은) 임시 파일은 즉시 삭제

### 6. 일기 삭제 (Delete)

삭제할 날짜의 파일을 선택

암호키 인증 성공 시, rm 명령어를 통해 파일이 영구적으로 삭제

--------------------------------------------------------------------------

## STEP 3. 데이터 보관하기 (백업 관리)

### 7. 일기 백업 (Backup)

생성, 수정, 삭제된 모든 변경 사항을 git commit으로 기록

git push 명령어를 통해 연결된 Github 원격 저장소로 데이터를 전송

### 8. 일기 동기화 (Sync)

다른 기기에서 작성된 일기가 있다면 git pull 명령어를 통해 최신 상태를 로컬(내 컴퓨터)로 가져옴.
